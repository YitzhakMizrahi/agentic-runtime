# Glossary

This glossary defines key terms used in the Agentic Runtime project.

---

## Agent

A goal-driven entity that coordinates reasoning and execution.

---

## Protocol

A reasoning process used by agents, such as planning or reflection.

---

## Runtime

The execution environment responsible for managing agent behavior and tool interactions.

---

## Context

The environment interface available to an agent.

It provides access to tools, system state, and memory.

---

## Tool

An executable capability that agents can invoke through the runtime.

Examples include file operations, Git commands, or API calls.

---

## Plan

A structured sequence of steps produced by the planner.

Plans describe how the agent will attempt to achieve a goal.

---

## Simulation

A pre-execution evaluation of a plan to detect potential issues.

---

## Execution

The stage in which the runtime invokes tools and performs actions.

---

## Reflection

A reasoning process that analyzes past actions to improve future behavior.

---

## Memory

A persistent store of agent experiences and knowledge.
